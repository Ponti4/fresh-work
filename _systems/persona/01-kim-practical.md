## 🎯 동기 & 목표

### Why (근본 동기)
```
"매일 같은 일을 반복하면서 정말 괴로움.
 이 시간을 줄일 수 있다면 더 큰 프로젝트에 집중할 텐데..."
```

- **핵심 문제**: 일은 많은데 자동화할 수 있는 도구가 없음
- **현재 상황**: 엑셀, 구글 시트로 수동 반복 작업
- **시간 낭비**: 주 10-15시간을 반복 업무에 소비
- **기술 배경**: 코딩은 해본 적 없지만, 학습 의지는 강함

### What (4주 후 바라는 결과)
```
"가장 반복되는 3개 업무를 자동화해서
 주 10시간을 아까운 작업에 쓸 수 있게 하고 싶다"
```

- **목표 1**: 일일 반복 작업 → 자동화 Skill 1개 완성
- **목표 2**: 월간 반복 작업 → 자동화 Skill 1개 완성
- **목표 3**: 데이터 정리 작업 → 자동화 Skill 1개 완성
- **기대 효과**: 주 5-10시간 시간 절약

### Success Criteria (성공의 정의)
| 지표 | 목표 | 측정 방법 |
|-----|------|---------|
| **Skill 개수** | 3개 완성 | 만든 Skill 개수 |
| **시간 절약** | 주 5시간 이상 | 자동화 전후 비교 |
| **안정성** | 1개월 무중단 운영 | 자동화된 작업 오류 0건 |
| **재사용성** | 최소 2-3번 이상 실제 사용 | 실제 사용 횟수 기록 |

## 📊 행동 패턴 & 선호도

### 학습 스타일
```
"5분 영상으로 한 가지 개념" > "50페이지 문서"
"따라하기 예제" > "이론 설명"
"이건 왜? 해야 하는가?" 질문 많음
```

- **영상 학습**: 선호 (한국어, 3-5분 단위)
- **텍스트 학습**: 요약 정도만 (전체 문서는 안 읽음)
- **손으로 하기**: 매우 중요 (직접 따라 해야 이해함)
- **즉시 피드백**: 원함 (언제 성공했는지 알고 싶음)

### 결과 지향성
- **과정 vs 결과**: 100% 결과 지향
- **어떤 과정인지**는 관심 없고, **결과가 나오는지**만 중요
- **미적 완성도**: 낮음 (작동하면 OK)
- **확장 가능성**: 고려 안 함 (지금 필요한 것만)

### 커뮤니티 & 독립성
- **혼자 vs 함께**: 처음엔 혼자이지만, 막히면 물어보기 원함
- **커뮤니티**: 관심 없음 (개인 시간 부족)
- **성과 공유**: 할 수는 있는데, 크게 관심 없음
- **경쟁 의식**: 약함 (동료 비교는 안 함)

## 💔 고통점 (Pain Points)

### 주요 고통점

**1. 시간 낭비 (최상위)**
```
"매일 50분을 쓰는 이메일 정리...
 이걸 5분으로 줄일 수 있다면 월 10시간 절약인데"
```
- 측정 가능한 반복 작업
- 명확한 개선 욕구

**2. 학습 진입장벽**
```
"코딩을 배워야 한다는데,
 Python이 뭐고 왜 해야 하는지 모르겠어"
```
- "Python 기초부터" 같은 말이 두려움
- "이걸 배우려면 6개월 걸린다"는 얘기가 무섭나
- "그냥 누가 만들어주면 좋겠는데" 심정

**3. 기술 부족감**
```
"내가 정말 할 수 있을까?
 컴퓨터 잘하는 사람들 것처럼..."
```
- 자신감 결여
- "또 뭘 설치해야 하나" 불안감
- 에러 메시지 봤을 때 당황

**4. 피드백 부족**
```
"이게 맞게 진행되고 있는 건가?
 내가 언제쯤 성공할 수 있는 거야?"
```
- 진도가 보이지 않으면 불안
- "이 수준이면 충분한 건가?" 의심

### 시도했던 해결책
- 엑셀 매크로 배우려다가 포기 (너무 복잡)
- IFTTT 써봤지만 제한이 많음
- 누군가에게 자동화 부탁했지만 비쌈 (月 20-50만원)

### 왜 실패했나
- **러닝커브가 가파름**: "Python 배우기"에 막힘
- **즉시성 부족**: 배우는 데 시간이 오래 걸림
- **손에 잡히는 결과 부족**: 몇 시간 해도 "뭔가 된" 게 없음
- **포기 신호**: 2-3주 안에 포기함

## 🎓 Kim을 위한 설계 원칙

### 1. 최소 기술 부담 (Minimal Tech Burden)
```
"일단 Python이 뭐고 왜 쓰는지는
 알 필요 없고, 이 명령어를 복사-붙여넣기 하세요"
```

**설계 방향**:
- ❌ "변수란 뭐고 함수는..." (설명 금지)
- ✅ "이 코드를 여기에 붙여넣고 엔터" (직접 실행)
- ✅ 기존 템플릿 사용 (처음부터 안 함)

### 2. 2-3일마다 완성된 결과 (Quick Wins)
```
"아, 이게 되네?" 느낌을 자주 경험
```

**설계 방향**:
- ✅ Day 1: 아이디어 명확 → 마크다운 문서 산출
- ✅ Day 2: 설계 완료 → 시각화된 설계 문서
- ✅ Day 3: 구현 → 실제 코드 파일
- ✅ Day 4: 테스트 → "작동합니다!" 메시지

### 3. 단계별 가이드 (Step-by-Step)
```
1. 명령어 복사
2. 엔터 누르기
3. 결과 확인
4. 다음 단계
```

**설계 방향**:
- ✅ 각 명령어가 정확함
- ✅ 예상 결과가 명확함
- ✅ 실패 시 "흔한 문제" 가이드

### 4. 직관적 피드백 (Clear Feedback)
```
✅ 완료됨
⏳ 진행 중 (50%)
❌ 실패 (이렇게 해결해보세요)
```
