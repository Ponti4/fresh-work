## 🎯 동기 & 목표

### Why (근본 동기)
```
"포트폴리오가 약해서 취업이 안 되는데,
 자동화 Skill을 만들면서 개발 경험도 쌓고
 포트폴리오도 채우고 싶어"
```

- **핵심 문제**: 경력이 없어서 취업이 어려움
- **현재 상황**: 학교 과제 정도만 했고, 실제 문제 해결 경험 부족
- **기대**: "이력서에 쓸 만한 프로젝트 3개 + 부수입 조금"
- **기술 배경**: Python 기초는 있지만, 실무는 모름

### What (4주 후 바라는 결과)
```
"실제로 작동하는 3개 자동화 도구를 만들어서
 포트폴리오에 넣고, GitHub에도 공유하고,
 경험담을 면접에서 얘기하고 싶다"
```

- **목표 1**: 음성→텍스트 변환 도구 완성 (학습 + 포트폴리오)
- **목표 2**: 데이터 크롤링 도구 완성 (실무 경험)
- **목표 3**: 웹 상의 자동화 서비스 배포 (추가 경험)
- **기대 효과**: 포트폴리오 강화 + 부수입 가능성

### Success Criteria
| 지표 | 목표 | 측정 방법 |
|-----|------|---------|
| **완성 프로젝트** | 3개 완성 | 만든 프로젝트 개수 |
| **GitHub 별** | 10+ 스타 | GitHub 저장소 추적 |
| **포트폴리오 품질** | 면접 질문 5개 이상 받음 | 면접에서 경험담 설명 |
| **배포** | 1개 이상 온라인 배포 | 실제 사용 가능한 서비스 |

## 🛠️ 제약사항 & 환경

### 기술적 제약
- ✅ Python 기초 있음 (학교에서 배움)
- ✅ Git 사용 가능
- ❌ 실무 경험은 0 (프로젝트 규모의 코드 못 짬)
- ❌ 배포/서버 경험 없음 (혼자 하기 싫음)
- ✅ 구글링 능력은 뛰어남 (스택오버플로우 자주 봄)

### 시간 & 환경
- **투자 가능 시간**: 주 10-15시간 (학기 중), 주 20+ (방학)
- **학습 선호**: "실제 프로젝트로 배우는 것" 선호
- **도움 요청**: 스스로 먼저 시도, 막혀도 쉽게 포기 안 함
- **네트워크**: 학교 또는 카페 (안정적 네트워크)

### 도구 & 접근성
- **장비**: 노트북 (i5-i7급, 8-16GB RAM)
- **GPU**: 있을 수도, 없을 수도 (상관없음)
- **비용**: 거의 쓸 수 없음 (무료 API 우대)
- **배포**: 무료 플랫폼 원함 (Heroku, Vercel, PythonAnywhere 등)

```
"인프런이나 유튜브로 배우는 거 좋아해.
 실제 프로젝트 코드 보면서 따라하는 게 제일 좋아"
```

- **프로젝트 기반**: 선호 (이론보다 실습)
- **커뮤니티**: GitHub, 깃허브 이슈 즐겨 봄
- **참고 자료**: 다른 좋은 프로젝트 코드 보는 거 좋아함

### 결과 지향성
- **과정 vs 결과**: 70% 결과 / 30% 과정
- **완성도**: 높음 (포트폴리오에 올릴 거라서)
- **확장 가능성**: 관심 있음 (나중에 더 확장할 수 있으면 좋음)
- **공유**: 매우 중요 (GitHub에 올리고, 블로그에 쓰기)

### 커뮤니티 & 독립성
- **혼자 vs 함께**: 혼자는 되지만 함께할 때 더 동기부여 됨
- **성과 공유**: 매우 함 (GitHub, 링크드인, 블로그)
- **경쟁 의식**: 있음 (다른 취준생들과 비교)
- **네트워킹**: 관심 있음 (취업에 도움이 될까봐)

## 💔 고통점 (Pain Points)

### 주요 고통점

**1. 포트폴리오 약함 (최상위)**
```
"면접 가서 "뭘 했어?" 물어보면
 "학교 과제요" 라고 답할 수밖에 없어.
 실제 문제를 푼 경험이 있으면 좋을 텐데"
```
- 학교 과제는 너무 간단함
- 실무 규모 프로젝트 경험 부족

**2. 취업 경쟁**
```
"지금 진짜 취업 어렵고,
 좋은 포트폴리오 없으면 면접도 못 본대"
```
- 경력 경쟁 심함
- 차별화 필요

**3. 배포 경험 부족**
```
"로컬에서는 되는데 온라인에 올릴 줄 몰라.
 Heroku 설정도 어렵고, AWS는 비싸고..."
```
- 배포는 배운 적 없음
- 부담스러움

**4. 부수입 필요**
```
"학비도 있고 생활비도 있는데,
 자동화 도구 팔면 돈 좀 벌 수 있을까?"
```
- 경제적 어려움
- 스킬 판매 가능성 있으면 좋음

### 시도했던 해결책
- 인프런 강의 여러 개 수강 (완강률 낮음)
- GitHub에 개인 프로젝트 올림 (너무 간단함)
- 알고리즘 문제 풀이 (이력서에 못 씀)
- 스타트업 인턴십 지원 (떨어짐)

### 왜 실패했나
- **완성의 어려움**: "완성도 있는 프로젝트"를 만들기 어려움
- **배포 복잡성**: 로컬에서 작동하는 것과 배포는 다름
- **비즈니스 모델 불명확**: 만들긴 했는데 어떻게 활용할지 모름
- **동기 지속성**: 혼자 하다가 포기하기 쉬움
