# Git Commit - 작업 내용 정리 및 커밋

**목표**: 변경 사항을 자동 분석하고, 커밋 메시지를 검토하며, 동의 후 커밋 실행

---

## 🎯 이 명령어는?

현재 작업한 내용을 **자동으로 분석**하고:
- ✅ **변경 내용 파악** (어떤 파일이 변경됐나?)
- 📝 **커밋 메시지 작성** (무엇을 했는가?)
- ✓ **동의 확인** (이 메시지로 커밋할까?)
- 🚀 **커밋 실행** (Git에 저장)

---

## 🚀 사용 방법

```bash
/git-commit
```

또는 작업 설명을 함께 제공:

```bash
/git-commit "기능 추가: 사용자 인증 시스템"
```

---

## 📋 진행 흐름

### Step 1: 변경 내용 자동 분석 (AI 작업)

당신이 명령어를 실행하면, Claude가 자동으로:
- `git status` 실행 → 변경된 파일 목록 확인
- `git diff` 실행 → 구체적인 변경 내용 확인
- `git log` 실행 → 최근 커밋 스타일 확인

**분석 결과 예시:**
```
📊 변경 사항 분석
- 수정된 파일: 3개
- 추가된 파일: 1개
- 삭제된 파일: 0개

변경 파일 목록:
✏️ src/auth.py (코드 수정)
✏️ tests/test_auth.py (테스트 추가)
✨ docs/AUTH.md (새 문서)
```

---

### Step 2: 작업 내용 확인 (사용자 입력)

**질문**: 이 변경 사항의 주요 내용을 한 문장으로 설명해주세요.

사용자가 제공하지 않은 경우:
- 명령어 인자에 설명이 있으면 그것을 사용
- 없으면 변경 파일 분석을 통해 Claude가 제안

**입력 예시:**
```
사용자 로그인 기능 구현 및 테스트 추가
```

---

### Step 3: 커밋 메시지 자동 생성

변경 내용과 커밋 스타일을 바탕으로 커밋 메시지를 작성합니다.

**생성된 메시지 예시:**
```
feat: 사용자 인증 시스템 구현

- 로그인/로그아웃 엔드포인트 추가
- JWT 토큰 기반 인증 로직 구현
- 인증 관련 테스트 케이스 작성
```

---

### Step 4: 커밋 메시지 확인 (사용자 동의)

**질문**: 이 커밋 메시지로 진행할까요?

**선택 사항:**
- ✅ 네, 커밋하세요 → Step 5로 진행
- ✏️ 메시지 수정 → 수정할 부분 입력 후 재생성
- ❌ 취소 → 커밋 중단

---

### Step 5: 커밋 실행

선택한 커밋 메시지로 Git 커밋을 실행합니다.

**실행 명령어:**
```bash
git add <변경된 파일들>
git commit -m "생성된 메시지"
```

**완료 메시지:**
```
✅ 커밋 완료!

커밋 정보:
- Hash: a1b2c3d
- Author: [사용자명]
- Date: 2025-01-12
- Message: feat(auth): 사용자 인증 시스템 구현
```

---

## 💡 핵심 특징

✅ **자동 분석** - 변경 내용을 직접 검사해서 파악
✅ **스타일 일관성** - 기존 커밋 메시지 패턴 추적
✅ **검증 단계** - 커밋 전에 메시지 확인
✅ **수정 가능** - 메시지가 마음에 안 들면 수정
✅ **안전성** - 실수로 커밋하는 것 방지

---

## 🔑 중요 규칙

### 커밋 메시지 형식

이 프로젝트는 **Conventional Commits** 스타일을 따릅니다:

```
<type>(<scope>): <subject>

<body>

Co-Authored-By: Claude Haiku 4.5 <noreply@anthropic.com>
```

**Type 예시:**
- `feat` - 새로운 기능
- `fix` - 버그 수정
- `docs` - 문서 수정
- `refactor` - 코드 리팩토링
- `test` - 테스트 추가/수정
- `chore` - 빌드, 의존성 등

**Scope 예시:**
- `auth` - 인증 관련
- `api` - API 관련
- `ui` - UI 관련

## ❓ 자주 묻는 질문

**Q. 커밋할 파일을 선택할 수 있나요?**
A. 먼저 `git add <파일>`로 스테이징한 후 명령어를 실행하세요.

**Q. 빈 커밋(변경 사항 없음)은 어떻게 하나요?**
A. 변경 사항이 없으면 경고 메시지를 표시하고 중단합니다.

**Q. 메시지를 완전히 다시 쓸 수 있나요?**
A. 네, Step 4에서 "메시지 수정" 선택 시 전체 수정 가능합니다.
