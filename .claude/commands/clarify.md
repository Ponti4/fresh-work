# Clarify - 자동화 요구사항 명확화

**목표**: 모호한 "반복 업무" → 구현 가능한 "자동화 Task 정의서"로 변환

---

## 🎯 이 명령어는?

자동화 아이디어를 **구현 전 가드레일**로 검증하는 대화형 명령어입니다.

5가지 질문으로:
- ✅ **정말 자동화할 만한가?** (빈도, 규칙성)
- ⚡ **문제의 본질이 무엇인가?** (극단화 분석)
- 🎯 **해결 후 어떤 가치가 있는가?** (기대 효과)

이 3가지를 명확히 한 후, Plan Mode에서 구현을 진행합니다.

---

## 🚀 사용 방법

```bash
/clarify "매일 하는 SNS 댓글 정리"
```

---

## 📋 진행 흐름

**⚠️ 중요**: 한 번에 하나의 질문만 받으니, 순서대로 답변해주세요.

### 1️⃣ 맥락 요약
당신의 반복 업무를 이해한 내용을 정리해서 보여줍니다.

### 2️⃣ Q1-Q5 순차 질문
- **Q1**: 빈도 (얼마나 자주?)
- **Q2**: 소요시간 (얼마나 오래?)
- **Q3**: 규칙성 (항상 같은 방식?)
- **Q4**: 극단화 (10배 늘면 뭐가 문제?)
- **Q5**: 가치도출 (해결되면 뭘 할?)


### 3️⃣ Task 정의서 생성
모든 답변을 바탕으로 간단한 Task 정의서를 만들고,
자동으로 `docs/_clarify/` 경로에 저장합니다.

### 4️⃣ Plan Mode로 진행
생성된 Task 정의서를 바탕으로 `/plan` 명령어로
구현 설계를 시작합니다.
