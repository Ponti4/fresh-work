# Clarify - 자동화 요구사항 명확화

**목표**: 모호한 "반복 업무" → 구현 가능한 "Task 정의서"로 변환

---

## 🎯 이 명령어는?

반복 업무를 **자동화하기 전에 명확히 정의**하는 대화형 명령어입니다.

한 번에 하나씩 질문하며:
- ✅ **불편함 감지** (가장 귀찮은 부분은?)
- ⚡ **기능 분해** (뭘 자동화할 건지?)
- 🎯 **최소 기능 정의** (이것만 해도 되지 않을까?)
- 📡 **외부 연동 확인** (추가 서비스 필요?)

이 과정을 거쳐 Plan Mode로 진행합니다.

---

## 🚀 사용 방법

```bash
/clarify "매일 하는 SNS 댓글 정리"
```

---

## 📋 진행 흐름

**⚠️ 중요**: 한 번에 하나의 질문만 받으니, 순서대로 답변해주세요.

### 1️⃣ 맥락 요약
당신의 반복 업무를 이해한 내용을 정리해서 보여줍니다.

### 2️⃣ Q1-Q6 순차 질문 (한 번에 하나씩)
- **Q1**: 빈도 (얼마나 자주?)
- **Q2**: 월 소요시간 (약 몇 시간?)
  → 연간 절감 시간 자동 계산
- **Q3**: 불편함 감지 (가장 귀찮은 부분?)
- **Q4**: 기능 분해 (뭘 자동화할 건지? / 최소한으로는?)
  → 3-4개 질문으로 MVP 정의
- **Q5**: 기대 효과 (해결되면 뭘 할?)
- **Q6**: 외부 연동 (Gmail, Slack 등 필요?)
  → 필요시 추가 논의

### 3️⃣ Task 정의서 생성
모든 답변을 바탕으로 간단한 Task 정의서를 만들고,
자동으로 `docs/_clarify/` 경로에 저장합니다.

### 4️⃣ Plan Mode로 진행
생성된 Task 정의서를 바탕으로 `/plan` 명령어로
구현 설계를 시작합니다.
