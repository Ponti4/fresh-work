# 개발 로그 생성 (DevLog Writer)

**현재 Claude Code 세션의 Context를 활용하여 개발 로그를 작성합니다.**

이 명령어는:
- 🧠 **현재 대화의 맥락**을 인식
- 📝 **5가지 핵심 질문**으로 당신의 의도를 파악
- 📂 **날짜별 독립 파일**로 자동 저장 (`docs/_devlog/devlog_YYYYMMDD_{제목}.md`)
- 🔄 **재사용 가능한 자산**으로 활용 (개인 회고 + 블로그 기초 자료)

---

## 🚀 사용법

```bash
/devlog
```

**흐름:**
1. 명령어 실행
2. 5개 질문에 **주관식으로 자유롭게** 답변
3. 자동 문서화 (`docs/_devlog/devlog_YYYYMMDD_{제목}.md`)
4. 필요시 추가 논의 또는 즉시 저장

---

## 📋 Interactive 질문 Flow

### **질문 1️⃣: 작업 제목/주제**
> 이번 세션에서 다룬 핵심 작업이 무엇인가요?

**예시:**
- "DevLog 시스템 개선"
- "AI 리터러시 템플릿 개발"
- "다중 도구 세션 통합 설계"

---

### **질문 2️⃣: 문제 상황 (Before)**
> 이 작업을 시작하게 된 배경이나 불편함이 무엇이었나요?

**예시:**
- "다중 도구 지원으로 자동화 복잡도가 높아짐"
- "세션 파일 파싱 후 수동 정리 과정이 번거로움"
- "기존 DEVLOG.md 누적으로 관리 어려움"

---

### **질문 3️⃣: Context의 역할**
> 이번 작업에서 Claude Code 세션이 어떤 역할을 했나요?

**예시:**
- "현재 대화를 직접 활용해 수동 작업을 감소시킴"
- "복잡한 아키텍처를 실시간으로 논의하며 설계함"
- "반복적인 개선 과정을 맥락 있게 기록할 수 있었음"

---

### **질문 4️⃣: DevLog의 목적**
> 이 DevLog를 어떻게 활용할 예정인가요?

**선택 옵션:**
- **A) 개인 회고용** - 자신의 성장과 변화를 추적하는 개인 기록
- **B) 팀/커뮤니티 공유용** - 팀원이나 같은 프리랜서들과 함께 배운 점 기록
- **C) 블로그/포스팅 기초 자료** - 나중에 `/ai-case` 명령어로 블로그 포스팅으로 변환
- **D) 포트폴리오 & 경력 자산** - 보유 기술, 해결 경험, 성과를 담아 나중에 포트폴리오로 활용

**기타 선택지가 있다면 주관식으로 입력해주세요**

---

### **질문 5️⃣: 주요 산출물**
> 이번 작업의 주요 결과물이나 생성된 파일이 있다면?

**예시:**
- `.claude/commands/devlog-writer.md` (명령어 정의)
- `docs/_devlog/` (DevLog 저장 경로)
- `_systems/01-devlog/` (통합 시스템 참조)

## 🔄 DevLog와 AI Case Writer 파이프라인

```
[현재 세션 작업]
         ↓
    /devlog 실행
    (5개 질문 답변)
         ↓
docs/_devlog/devlog_YYYYMMDD_{제목}.md 생성
    (개인 회고, 팀 공유, 포트폴리오용)
         ↓
[선택사항] /ai-case 실행
(위 DevLog를 분석하여 블로그 포스팅 변환)
         ↓
.claude/commands/gpters-ai-case-writer.md 로직 따라
블로그 포스팅 초안 생성
         ↓
게시 또는 보존
```

**즉, DevLog는:**
- ✅ **1차 목적: 현재 작업의 맥락과 배움을 기록** (개인/팀)
- ✅ **2차 목적: 블로그 변환의 기초 자료** (선택사항)

## 💭 DevLog의 가치

이 DevLog는 단순한 **작업 기록**이 아닙니다:

1. **자산화** - 개인 회고로 성장을 추적하고, 블로그 기초 자료로 재활용
2. **맥락 보존** - 복잡한 결정 과정과 이유를 남겨서 나중에 참고
3. **AI 활용 리터러시** - 비개발자도 AI와 함께 일하는 방식을 학습
4. **다음 자동화의 아이디어** - 이전 경험을 바탕으로 더 나은 자동화 설계
