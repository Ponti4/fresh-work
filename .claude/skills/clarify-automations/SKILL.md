# Clarify Automations - 요구사항 명확화 스킬

## 🎯 역할

자동화 아이디어를 **구현 가능한 Task 정의서**로 변환하는 대화형 스킬입니다.

프리랜서가 반복 업무를 제시하면, 5가지 단계별 질문을 통해:
- ✅ **정말 자동화할 만한가?** (빈도, 규칙성 검증)
- ⚡ **문제의 본질이 무엇인가?** (극단화 사고)
- 🎯 **해결 후 어떤 가치가 있는가?** (가드레일)

이 3가지를 명확히 한 후, Plan Mode 진행 전 **최종 가드레일** 역할을 합니다.

---

## 📋 핵심 원칙

**⚠️ 원격 제어 원칙 (Strictly One-by-One)**

1. 모든 질문은 **한 번에 하나씩만** 던집니다.
2. 사용자의 답변이 올 때까지 다음 질문으로 절대 넘어가지 않습니다.
3. 한 번의 응답에는 **하나의 질문 번호(Q1, Q2 등)**만 포함됩니다.

---

## 🚀 워크플로우

### 📍 Step 1: 맥락 요약 (사용자 입력 수신)

사용자가 `/clarify "반복 업무 설명"`으로 입력하면:

먼저 사용자의 입력을 요약하고 예상 흐름을 제시합니다.

```
🤖 AI가 이해한 맥락 (3줄)
[사용자 입력 요약]

🌊 예상 Userflow (5단계)
1. 현상 파악 (빈도, 소요시간, 규칙성)
2. 극단화 분석 (문제의 본질)
3. 가치 도출 (해결 후 기대효과)
4. Task 정의
5. 문서화 및 저장
```

---

### 📍 Step 2: Phase 1 - 현상 파악 (Q1, Q2, Q3)

**Q1: 빈도**
> "이 업무를 얼마나 자주 하시나요?"
- 1. 매일 (주 5회 이상)
- 2. 자주 (주 2-4회)
- 3. 가끔 (월 1-3회)
- 4. 드물게 (분기 1회 미만)

**[사용자 응답 대기]**

---

**Q2: 소요시간**
> "한 번 할 때 보통 얼마나 걸리나요?"
- 1. 5분 미만
- 2. 30분 내
- 3. 1시간 내
- 4. 1시간 이상

**[사용자 응답 대기]**

---

**Q3: 규칙성**
> "매번 거의 동일한 방식으로 하시나요?"
- 예 / 아니오

**[사용자 응답 대기]**

---

### 📍 Step 3: Phase 2 - 극단화 (Q4)

**Q4: 극단화 (문제의 본질)**
> "만약 이 업무가 내일부터 10배로 늘어난다면, 무엇이 가장 큰 문제일까요?"
- 1. 시간 부족
- 2. 실수 폭발
- 3. 멘탈 붕괴
- 4. 포기하고 싶음

**[사용자 응답 대기]**

---

### 📍 Step 4: Phase 3 - 가치 도출 (Q5)

**Q5: 목표 (기대 효과)**
> "이 문제가 완벽히 해결되어 여유 시간이 생긴다면, 무엇을 하시겠습니까?"
- 1. 휴식
- 2. 자기계발
- 3. 본업 집중
- 4. 칼퇴

**[사용자 응답 대기]**

---

### 📍 Step 5: Task 정의서 생성 및 저장

모든 질문 답변 수집 후:

1. **Task 정의서 작성** (간단한 형식)
   - 업무명, 빈도, 월 소요시간, 핵심 불편함, 기대 효과

2. **자동 저장**
   - 경로: `docs/_clarify/clarify_YYYYMMDD_{title}.md`
   - 형식: 마크다운

3. **다음 단계 안내**
   - Plan Mode 진행 제안

## 📤 산출물 형식

See: [output-template.md](./references/output-template.md)

예시:
```markdown
# Task 정의서

## 📌 개요
- **업무명**: SNS 댓글 정리
- **작성일**: 2025-01-05

## 🔍 현황 분석
- **빈도**: 매일 (주 5회)
- **소요시간**: 한 번에 30분
- **규칙성**: 예
- **월 소요시간**: 약 40시간

## ⚡ 핵심 불편함
시간 부족으로 다른 업무 진행 어려움

## 🎯 기대 효과
본업 집중 시간 확보

## ✅ 다음 단계
`/plan` 모드에서 자동화 구현 설계 진행
```

## 🛠️ 사용 방법

### 명령어 호출
```bash
/clarify "반복 업무 설명"
```

### 흐름
1. 사용자가 명령어 입력 (반복 업무 설명)
2. 맥락 요약 제시
3. Q1-Q5 순차 질문 (한 번에 하나씩)
4. 각 답변 수집
5. Task 정의서 자동 생성
6. `docs/_clarify/clarify_YYYYMMDD_{title}.md` 저장

## 💡 핵심 지침

### 비개발자 친화
- 전문 용어 배제
- 쿠션어 사용 ("괜찮으시면", "혹시", "혹은" 등)
- 이해하기 쉬운 예시 제시

### 순차적 심층 문답
- 한 번에 하나의 질문만
- 선택지 제시 (자유 입력 아님)
- 사용자 응답을 적극 인정하고 반영

### 문제의 본질 파악
- Q4 극단화는 핵심 (시간/오류/정신건강 중 우선순위 파악)
- Q5 가치도출은 가드레일 (정말 필요한가? 동기 확인)

## ✨ 특징

✅ **가드레일 역할** - Plan Mode 전 최종 검증
✅ **비전공자 친화** - 선택지 기반, 쉬운 언어
✅ **자동 문서화** - 답변 내용 자동 저장
✅ **재사용 가능** - 여러 자동화 Task 반복 적용
