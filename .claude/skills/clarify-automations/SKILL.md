# Clarify Automations - 요구사항 명확화 스킬

## 🎯 역할

반복 업무를 **명확한 자동화 Task로 정의**하는 대화형 스킬입니다.

프리랜서가 반복 업무를 제시하면, 6가지 단계별 질문을 통해:
- ✅ **불편함 감지** (가장 귀찮은 부분은?)
- ⚡ **기능 분해** (뭘 자동화할 건지?)
- 🎯 **최소 기능 정의** (MVP로 충분한가?)
- 📡 **외부 연동 확인** (추가 서비스 필요?)

이 과정을 거쳐 **Plan Mode에서 상세 설계**를 진행합니다.

---

## 📋 핵심 원칙

**⚠️ 원격 제어 원칙 (Strictly One-by-One)**

1. 모든 질문은 **한 번에 하나씩만** 던집니다.
2. 사용자의 답변이 올 때까지 다음 질문으로 절대 넘어가지 않습니다.
3. 한 번의 응답에는 **하나의 질문 번호(Q1, Q2 등)**만 포함됩니다.

---

## 🚀 워크플로우

### 📍 Step 1: 맥락 요약 (사용자 입력 수신)

사용자가 `/clarify "반복 업무 설명"`으로 입력하면:

먼저 사용자의 입력을 요약하고 예상 흐름을 제시합니다.

```
🤖 AI가 이해한 맥락 (3줄)
[사용자 입력 요약]

🌊 예상 Userflow (6단계)
1. 불편함 감지 (가장 귀찮은 부분)
2. 기능 분해 (뭘 자동화할 건지)
3. 최소 기능 정의 (이것만 해도 되나)
4. 기대 효과 (해결 후 기대)
5. 외부 연동 확인 (추가 서비스 필요)
6. Task 정의서 생성 및 저장
```

---

### 📍 Step 2: Phase 1 - 수치 파악 (Q1, Q2)

**Q1: 빈도**
> "이 업무를 얼마나 자주 하시나요?"
- 1. 매일 (주 5회 이상)
- 2. 자주 (주 2-4회)
- 3. 가끔 (월 1-3회)
- 4. 드물게 (분기 1회 미만)

**[사용자 응답 대기]**

---

**Q2: 소요시간**
> "한 번 할 때 보통 얼마나 걸리나요?"
- 1. 5분 미만
- 2. 30분 내
- 3. 1시간 내
- 4. 1시간 이상

**계산**: 월 회수 × 한 번 소요시간 = **월 약 X시간**
→ 연간 절감 시간 자동 계산

**[사용자 응답 대기]**

---

### 📍 Step 3: Phase 2 - 불편함 및 기능 (Q3, Q4)

**Q3: 불편함 감지**
> "이 업무에서 가장 귀찮고 힘든 부분이 뭔가요?"
- (사용자 자유 답변)
- 예: "데이터 분류", "수동 입력", "반복되는 체크"

**[사용자 응답 대기]**

---

**Q4: 기능 분해 (MVP 정의)**

Q3 답변을 바탕으로 **3-4개 질문 반복**:

1. "그렇다면 우리가 자동화해야 할 핵심은?"
   - (사용자 답변)

2. "이것만 해도 충분할까요, 아니면 더 필요한가요?"
   - (사용자 답변 → 스코프 명확화)

3. "더 간단하게 할 방법은 없을까요?"
   - (사용자 답변 → 복잡도 최소화)

→ **최종 확정**: "그러면 [기능]을 자동화하겠습니다"

**[사용자 응답 대기]**

---

### 📍 Step 4: Phase 3 - 기대 효과 (Q5)

**Q5: 목표 (기대 효과)**
> "이 기능이 자동화되면, 월 X시간을 확보하게 되는데 뭘 하시겠습니까?"
- 1. 휴식
- 2. 자기계발
- 3. 본업 집중
- 4. 칼퇴

**[사용자 응답 대기]**

---

### 📍 Step 5: Phase 4 - 외부 연동 확인 (Q6)

**Q6: 외부 연동 필요**
> "이 기능을 구현하기 위해 Gmail, Slack, Google Sheet 등 **외부 서비스 연동**이 필요할까요?"

- **아니오** → Plan Mode로 바로 진행
- **예** → "어떤 서비스인가요?"
  → 추가 논의 후 Plan Mode 진행

**[사용자 응답 대기]**

---

### 📍 Step 6: Task 정의서 생성 및 저장

모든 질문 답변 수집 후:

1. **Task 정의서 작성** (간단한 형식)
   - 업무명, 빈도, 월 소요시간, 핵심 불편함, 기대 효과

2. **자동 저장**
   - 경로: `docs/_clarify/clarify_YYYYMMDD_{title}.md`
   - 형식: 마크다운

3. **다음 단계 안내**
   - Plan Mode 진행 제안

## 📤 산출물 형식

See: [output-template.md](./references/output-template.md)

예시:
```markdown
# Task 정의서

## 📌 개요
- **업무명**: SNS 댓글 분류
- **작성일**: 2025-01-06

## 📊 현황
- **빈도**: 매일 (주 5회)
- **월 소요시간**: 약 6.5시간
- **연간 절감**: 약 78시간

## ⚡ 핵심 불편함
매일 아침 1시간 동안 댓글을 수동으로 분류하느라 다른 업무 진행 어려움

## 🎯 자동화할 기능
- 댓글을 카테고리(긍정/부정/일반)로 자동 분류
- 분류된 댓글을 각 폴더로 정렬

## 🎁 기대 효과
본업(영상 편집) 집중 시간 확보

## 📡 외부 연동
아니오 - Claude Code만으로 가능

## ✅ 다음 단계
`/plan` 모드에서 구현 설계 진행
```

## 🛠️ 사용 방법

### 명령어 호출
```bash
/clarify "반복 업무 설명"
```

### 진행 흐름
1. 사용자가 명령어 입력 (반복 업무 설명)
2. 맥락 요약 제시
3. Q1-Q6 순차 질문 (한 번에 하나씩)
   - Q1: 빈도
   - Q2: 월 소요시간 (연간 절감 계산)
   - Q3: 불편함 감지
   - Q4: 기능 분해 (3-4개 반복 질문)
   - Q5: 기대 효과
   - Q6: 외부 연동 여부
4. Task 정의서 자동 생성
5. `docs/_clarify/clarify_YYYYMMDD_{title}.md` 저장
6. `/plan` 명령어로 상세 설계 진행 (자세한 구현은 여기서)

## 💡 핵심 지침

### 비개발자 친화
- 전문 용어 배제
- 쿠션어 사용 ("괜찮으시면", "혹시", "혹은" 등)
- 이해하기 쉬운 예시 제시

### 순차적 심층 문답
- **한 번에 하나의 질문만** (원격 제어 원칙)
- 선택지 제시 (Q1-Q2, Q5 제외 Q3는 자유)
- 사용자 응답을 적극 인정하고 반영

### Q4 기능 분해의 목표
- MVP(최소 기능)로 스코프 정의
- 3-4개 반복 질문으로 명확화
- 복잡도 최소화 ("더 간단하게 할 방법은?")
- 문서화: 답변 내용 그대로 기록

### Q6 외부 연동
- 예/아니오만 확인
- 예인 경우: 서비스명만 수집 → 추가 논의
- 상세한 API 연동은 Plan Mode에서

## ✨ 특징

✅ **명확한 요구사항 정의** - 불편함 → 기능 → MVP로 명확화
✅ **비전공자 친화** - 한 번에 하나씩, 쉬운 언어
✅ **자동 문서화** - 답변 내용 자동 저장
✅ **Plan Mode와 연계** - 상세 설계는 Plan Mode에서
✅ **재사용 가능** - 3개 자동화를 4주에 완성할 수 있도록
