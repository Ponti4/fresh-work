# 10.3. Task명확화 - 6가지 질문

> **핵심 목표**: 월 소요시간 계산 + 자동화 ROI 판단 + MVP 기능 정의

---

## Phase 1: 현상 파악 (Fact Check)

### Q1: 빈도 - "이 업무를 얼마나 자주 하시나요?"

**선택지:**
| # | 선택지 | 빈도 | 월 회수 |
|---|--------|------|--------|
| 1 | **매일** | 주 5회 이상 | 20회 |
| 2 | **자주** | 주 2-4회 | 10회 |
| 3 | **가끔** | 월 1-3회 | 2회 |
| 4 | **드물게** | 분기 1회 미만 | 0.5회 |

**자동화 ROI 판단:**
- 1, 2번: ✅ 자동화 유망 (반복성 높음)
  - 월 20회 또는 월 10회 = 충분한 빈도로 자동화 가치 있음
- 3번: ⚠️ 검토 필수 (Q2 소요시간과 함께 고려)
  - 월 2회 = 반복성 낮음, 소요시간이 충분히 커야 함
- 4번: ❌ 자동화 비추 (수동 수행이 더 효율적)
  - 월 0.5회 = 거의 자동화 효과 없음

**다음 진행:**
```
감사합니다! 매일 반복하시는 업무이군요. (또는 "자주 하시는군요", "가끔 하시는군요")
```

---

### Q2: 소요시간 - "한 번 할 때 보통 얼마나 걸리나요?"

**선택지:**
| # | 선택지 | 한 번 소요시간 | 평균값 |
|---|--------|--------------|--------|
| 1 | **5분 미만** | ~5분 | 5분 |
| 2 | **30분 내** | 5-30분 | 20분 |
| 3 | **1시간 내** | 30분-1시간 | 40분 |
| 4 | **1시간 이상** | 60분 이상 | 90분 |

**월 소요시간 자동 계산 (Q1과 조합):**

| Q1 \ Q2 | 5분미만 | 30분내 | 1시간내 | 1시간이상 |
|---------|--------|--------|--------|----------|
| **매일** (20회) | ~1.5h/월 | ~6.5h/월 | ~13h/월 | ~30h/월 |
| **자주** (10회) | ~0.8h/월 | ~3.3h/월 | ~6.5h/월 | ~15h/월 |
| **가끔** (2회) | - | ~0.7h/월 | ~1.3h/월 | ~3h/월 |
| **드물게** (0.5회) | - | - | - | ~1.5h/월 |

**자동화 ROI 판단:**
- **월 5시간 이상** → 자동화 강력 추천 ✅✅
  - 연간 약 60시간 절감 (약 7.5일)
  - 자동화 개발 비용 대비 명확한 수익
- **월 3-5시간** → 자동화 추천 (규칙성 필수) ✅
  - 연간 약 36-60시간 절감
  - Q3 규칙성 높을 시 우선 추진
- **월 1-3시간** → 조건부 검토 ⚠️
  - 연간 약 12-36시간 절감
  - Q3 규칙성이 매우 높아야 가치 있음
- **월 1시간 미만** → 자동화 비추 ❌
  - 개발 비용 대비 효과 미미

**실제 계산 예시:**
```
Q1: 매일 (20회/월) + Q2: 30분내 (20분 평균)
→ 월 소요시간 = 20회 × 20분 = 400분 ≈ 6.5시간
→ 연간 절감 = 6.5시간 × 12개월 = 78시간

Q1: 자주 (10회/월) + Q2: 1시간내 (40분 평균)
→ 월 소요시간 = 10회 × 40분 = 400분 ≈ 6.5시간
→ 연간 절감 = 6.5시간 × 12개월 = 78시간
```

**다음 진행:**
```
감사합니다! 월 약 X시간이 소요되시는군요. 연간 Y시간을 절감할 수 있겠네요.
```

### Q3: 규칙성 - "매번 거의 동일한 방식으로 하시나요?"

**선택지:**
| 응답 | 의미 | 규칙성 |
|------|------|--------|
| **예** | 매번 거의 동일한 방식 | 높음 |
| **아니오** | 매번 다양한 방식 | 낮음 |

**규칙성에 따른 자동화 영향:**

| 응답 | 의미 | 자동화 난이도 | 성공 가능성 | 다음 단계 |
|------|------|--------------|-----------|---------|
| **예** | 패턴 반복 | 낮음 | 높음 ✅✅ | Q4로 진행 (표준화) |
| **아니오** | 매번 다름 | 높음 | 중간 ⚠️ | Q4 기능 분해 (극단화) |

**예시:**

**규칙성 높음 (예):**
- "매일 아침 SNS 댓글을 수집해서 엑셀에 정리" → 같은 방식 ✅
- "매주 월요일 보고서 생성 후 이메일 전송" → 같은 절차 ✅
- "신입 지원서를 일정한 기준으로 분류" → 기준 동일 ✅

**규칙성 낮음 (아니오):**
- "클라이언트 요청 처리" → 매번 다른 요청 ⚠️
- "긴급 이슈 대응" → 상황마다 다름 ⚠️
- 하지만 Q4 기능 분해로 "공통 부분만" 자동화 가능

**Q1-Q3 종합 판단:**
```
매일 + 30분 이상 + 규칙성 높음 = 자동화 강력 추천 ✅✅✅
매일 + 5-30분 + 규칙성 높음 = 자동화 추천 ✅✅
자주 + 30분 이상 = 조건부 추천 ✅
가끔 + 1시간 이상 = 매우 조건부 ⚠️
규칙성 낮음 = Q4에서 핵심 기능만 재정의 필요
```

**주의사항:**
- 규칙성이 낮아도 포기하지 마세요!
- Q4에서 "공통 부분"을 찾으면 극단화로 자동화 가능
- 예: 매번 다른 클라이언트 요청 → "요청 수집 및 초기 분류"만 자동화

**다음 진행:**
```
감사합니다! 규칙성이 높으시군요. (또는 "낮으시군요. 그래도 핵심 부분을 찾아보겠습니다.")
```

## Phase 2: 기능 분해 - 불편함부터 MVP까지

### Q4: 불편함 감지 & 기능 분해 - "가장 귀찮은 부분과 자동화할 기능은?"

> **목표:** 사용자의 pain point에서 출발해서 구현 가능한 MVP 기능으로 축소

**Step 1: 불편함 감지 (Pain Point 파악)**
> "이 업무에서 가장 귀찮고 힘든 부분이 뭔가요?"

**답변 방식:** 자유 입력 (선택지 없음)

**예상 답변 패턴:**
| 답변 유형 | 예시 | 자동화 기회 |
|---------|------|-----------|
| **데이터 처리** | "데이터 분류가 반복되는데 손으로 일일이 해야 함" | 분류 규칙 자동화 |
| **정보 수집** | "여러 곳에서 정보를 모아야 함" | 데이터 통합 |
| **수동 입력** | "수동 입력이 너무 많음" | 자동 입력 스크립트 |
| **체크 리스트** | "반복되는 체크 리스트 작성" | 템플릿 자동 생성 |
| **형식 변환** | "다양한 형식으로 변환해야 함" | 포맷 자동 변환 |

**활용:** 사용자가 느끼는 pain point를 그대로 포착하여 Q4의 맥락으로 사용

---

**Step 2: 핵심 기능 정의 (What to Automate)**
> "그렇다면 우리가 자동화해야 할 핵심 기능은 뭘까요?"

**답변 방식:** 자유 입력 (구체적인 액션)

**좋은 답변 vs 모호한 답변:**
| 좋은 답변 | 모호한 답변 | 개선 방향 |
|---------|----------|---------|
| "이메일을 자동으로 분류하기" | "이메일 관리" | "분류하는" 등 동사 포함 |
| "엑셀에 고객 데이터 자동 입력" | "고객 데이터 정리" | 대상과 방식 명확화 |
| "매일 수집한 댓글을 구글시트에 정리" | "댓글 정리" | 출처와 대상 명시 |

**예시:**
- "이메일을 자동으로 분류하고 정렬하기"
- "여러 소스의 데이터를 자동으로 모아 한곳에 정리하기"
- "엑셀에 자동으로 데이터 입력하기"
- "일일 보고서를 자동으로 생성해서 이메일로 전송"

---

**Step 3: 스코프 확정 (Is This Enough?)**
> "이것만 해도 충분할까요, 아니면 더 필요한가요?"

**목표:** 범위 명확화

**사용자 답변 처리:**
- **"네, 이것만 해도 충분합니다"** → Step 4로 진행
- **"아니, 추가로 [기능]도 필요합니다"**
  - 추가 기능을 명확히 확인
  - Step 4에서 모두 처리 가능한지 판단
  - 필요시 우선순위 결정

**과도한 스코프 감지:**
- 사용자가 너무 많은 기능을 나열하면?
  - "좋습니다. 이 중에서 가장 급한 것 2-3개부터 시작할까요?"
  - Step 4에서 우선순위 정리

---

**Step 4: 복잡도 최소화 (MVP 정의)**
> "더 간단하게 할 방법은 없을까요?"

**목표:** 스코프를 MVP 수준으로 축소 (파레토 법칙 80/20 적용)

**축소 기법:**

| 기법 | 예시 | 효과 |
|------|------|------|
| **범위 축소** | "모든 카테고리 분류" → "3개 카테고리만" | 개발 70% 감소 |
| **빈도 축소** | "실시간 처리" → "일일 배치" | 복잡도 50% 감소 |
| **대상 축소** | "전체 지메일" → "특정 발신자 메일" | 테스트 용이 |
| **기간 축소** | "과거 모든 데이터" → "최근 7일" | 성능 개선 |
| **기능 축소** | "분류+정렬+통계" → "분류만" | 일단 완성 후 확장 |

**예시:**
- "모든 카테고리 분류" → "3개 카테고리만 자동 분류, 나머지는 수동"
- "실시간 처리" → "매일 자정에 일괄 처리"
- "전체 SNS 모니터링" → "특정 계정의 댓글만 수집"
- "완전 자동" → "초기 분류는 자동, 최종 확인은 수동"

**최종 확정:**
```
감사합니다! 그러면 [최종 기능]을 자동화하겠습니다.
예: "매일 Gmail에서 특정 발신자 메일을 수신하면, 제목에 따라 3개 폴더로 자동 분류하기"
```

**기록 방식:**
- 사용자의 답변을 원문 그대로 기록
- 불편함 → 기능 → 최종 MVP의 흐름을 문서화

## Phase 3: 가치 도출 (Value)

### Q5: 목표 - "이 문제가 완벽히 해결되어 여유 시간이 생긴다면?"

**질문:**
> "자동화되어 월 X시간이 절감된다면, 그 시간으로 무엇을 하시겠습니까?"

**선택지:**

| # | 선택지 | 설명 | 신호 |
|----|--------|------|------|
| **1** | **휴식** | 개인시간 확보, 피로 회복 | ✅ 건강상 필요 |
| **2** | **자기계발** | 스킬 향상, 공부, 자격증 | ✅✅ 장기 가치 |
| **3** | **본업 집중** | 핵심 업무에 더 투자, 매출 증대 | ✅✅✅ 즉각적 ROI |
| **4** | **칼퇴** | 퇴근 시간 단축, 워라밸 | ✅ 삶의 질 |

**가드레일 분석 (동기와 신호):**

| 선택 | 사용자 상태 | 긍정적 신호 | 주의사항 |
|------|-----------|-----------|---------|
| **1. 휴식** | 피로도 높음 | 건강상 중요 ✅ | 과도한 피로는 번아웃 신호 |
| **2. 자기계발** | 성장 동기 높음 | 장기적 경력 개선 ✅✅ | 명확한 학습 목표 있는가? |
| **3. 본업 집중** | 사업/직무 열정 | 명확한 ROI ✅✅✅ | 월 X시간으로 실제 매출 증대 가능? |
| **4. 칼퇴** | 워라밸 중시 | 삶의 질 ✅ | 조직 문화와 맞는가? |

**해석:**
- **1번 선택:** "지금 정말 피로해하는 상태. 먼저 휴식이 중요!"
- **2번 선택:** "성장하려는 마음. 4주 목표와 연계하면 좋음"
- **3번 선택:** "명확한 비즈니스 동기. 자동화의 직접적 가치 증명"
- **4번 선택:** "워라밸 추구. 조직 문화 평가 지표 포함"

**Q5의 역할:**
- 자동화의 **진정한 동기** 파악
- 절감된 시간을 어떻게 활용할 것인가?
- CLAUDE.md 프로필의 "4주 목표"와 연계
- 사용자의 우선순위와 가치관 이해

**예시 대화:**

```
Claude: "좋습니다! 월 약 6.5시간이 절감되면, 그 시간으로 뭘 하고 싶으세요?"

사용자: "Python 공부하고 싶어요"
Claude: "정말 좋네요! 월 6.5시간이면 주당 1.5시간 정도.
         4주 내에 간단한 자동화 스크립트 정도는 학습할 수 있겠어요.
         그것도 이번 자동화 프로젝트에서 실전으로 배울 수 있으니 일석이조군요!"
```

**다음 진행:**
```
감사합니다! 월 약 X시간을 [선택지]에 투자하시겠군요.
자동화 후 그 시간으로 [목표]를 이루실 수 있을 것 같습니다.
```

## Phase 4: 외부 연동 확인

### Q6: 외부 연동 필요 - "추가 서비스 연동이 필요할까요?"

**질문:**
> "이 기능을 구현하기 위해 Gmail, Slack, Google Sheet, Airtable, Zapier, GitHub 등 **외부 서비스 연동**이 필요할까요?"

**선택지:**

| # | 선택지 | 설명 | 영향 |
|----|--------|------|------|
| **1** | **아니오** | Claude Code만으로 충분 | 단순함 ✅ |
| **2** | **예** | 외부 API/서비스 연동 필요 | 복잡도 증가 ⚠️ |

**외부 서비스 연동이 필요한 경우:**

**Step 1: 서비스명 수집**
> "그렇다면 어떤 서비스와 연동이 필요한가요?"

**예상 서비스 목록:**

| 카테고리 | 서비스 | 용도 | 난이도 |
|---------|--------|------|--------|
| **이메일** | Gmail, Outlook | 메일 수집/분류/전송 | 중간 |
| **채팅** | Slack, Discord | 알림, 메시지 전송 | 중간 |
| **스프레드시트** | Google Sheet, Excel | 데이터 기록, 리포트 | 낮음 |
| **데이터베이스** | Airtable, Notion | 데이터 저장, 검색 | 높음 |

## 📊 자동 계산: 월 소요시간 → 연간 절감

**계산식:**

```
빈도별 월 회수:
- 매일 (주5일) → 20회/월
- 자주 → 10회/월 (평균)
- 가끔 → 2회/월 (평균)
- 드물게 → 0.5회/월

한 번 소요시간 (평균):
- 5분미만 → 5분
- 30분내 → 20분
- 1시간내 → 40분
- 1시간이상 → 90분

월 소요시간 = 월 회수 × 한 번 소요시간
연간 절감 시간 = 월 소요시간 × 12개월
```

**예시 1: 매일 + 30분내**
```
Q1: 매일 (20회/월)
Q2: 30분내 (20분 평균)

월 소요시간 = 20회 × 20분 = 400분 ≈ 6.5시간
연간 절감 = 6.5 × 12 = 78시간 (약 10일)
```

**예시 2: 자주 + 1시간내**
```
Q1: 자주 (10회/월)
Q2: 1시간내 (40분 평균)

월 소요시간 = 10회 × 40분 = 400분 ≈ 6.5시간
연간 절감 = 6.5 × 12 = 78시간
```
