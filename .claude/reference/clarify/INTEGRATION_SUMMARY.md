## Q1-Q6 통합 현황

### Q1: 빈도 (이 업무를 얼마나 자주 하시나요?)

**추가 요소:**
- ✅ 선택지를 표 형식으로 재구성 (# | 선택지 | 빈도 | 월 회수)
- ✅ 각 선택지별 월 회수 명시 (20회, 10회, 2회, 0.5회)
- ✅ 자동화 ROI 판단 기준 상세화 (선택지별 설명 추가)
- ✅ 실제 값 예시 포함 (월 20회 = 반복성 높음)

**포함 내용:**
```
| # | 선택지 | 빈도 | 월 회수 |
|---|--------|------|--------|
| 1 | **매일** | 주 5회 이상 | 20회 |
| 2 | **자주** | 주 2-4회 | 10회 |
| 3 | **가끔** | 월 1-3회 | 2회 |
| 4 | **드물게** | 분기 1회 미만 | 0.5회 |
```

---

### Q2: 소요시간 (한 번 할 때 보통 얼마나 걸리나요?)

**추가 요소:**
- ✅ 선택지를 표 형식으로 재구성 (# | 선택지 | 한 번 소요시간 | 평균값)
- ✅ 예상 범위와 평균값 분리 (예: 5-30분 | 20분)
- ✅ 실제 계산 예시 2개 추가 (매일+30분, 자주+1시간)
- ✅ 월 소요시간 → 연간 절감 자동 계산 가이드
- ✅ ROI 판단 기준 상세화 (연간 시간 + 비용 효율성)

**추가된 계산 예시:**
```
Q1: 매일 (20회/월) + Q2: 30분내 (20분 평균)
→ 월 소요시간 = 20회 × 20분 = 400분 ≈ 6.5시간
→ 연간 절감 = 6.5시간 × 12개월 = 78시간
```

---

### Q3: 규칙성 (매번 거의 동일한 방식으로 하시나요?)

**추가 요소:**
- ✅ 선택지 표 형식 재구성
- ✅ 규칙성 높음/낮음에 따른 상세 예시 (3개씩)
- ✅ "규칙성 낮아도 포기하지 말기" 주의사항 추가
- ✅ 극단화 전략 예시 (클라이언트 요청 → 요청 수집만 자동화)
- ✅ Q1-Q3 종합 판단 기준 유지

**추가된 예시:**
```
규칙성 높음 (예):
- "매일 아침 SNS 댓글을 수집해서 엑셀에 정리" → 같은 방식
- "신입 지원서를 일정한 기준으로 분류" → 기준 동일

규칙성 낮음 (아니오):
- "클라이언트 요청 처리" → 매번 다른 요청
- → Q4 기능 분해로 "요청 수집 및 초기 분류"만 자동화
```

---

### Q4: 불편함 감지 & 기능 분해

**추가 요소 (4단계 구조 상세화):**

**Step 1: 불편함 감지**
- ✅ 자유 입력 방식 명시
- ✅ 예상 답변 패턴 표 (데이터 처리/정보 수집/수동 입력/체크리스트/형식변환)
- ✅ 자동화 기회별 연결 (분류 규칙 자동화, 데이터 통합 등)

**Step 2: 핵심 기능 정의**
- ✅ "좋은 답변 vs 모호한 답변" 비교표 추가
- ✅ 동사 포함, 대상/방식 명확화 예시
- ✅ 4가지 실제 예시 (이메일 분류, 데이터 통합, 엑셀 입력, 일일 보고서)

**Step 3: 스코프 확정**
- ✅ 사용자 답변 처리 방법 (네/아니오 분기)
- ✅ "과도한 스코프 감지" 방법 (너무 많은 기능 나열 시 대응)

**Step 4: 복잡도 최소화 (MVP)**
- ✅ 파레토 법칙 적용 명시
- ✅ 축소 기법 5가지 (범위/빈도/대상/기간/기능 축소)
- ✅ 각 기법별 효과 정량화 (70% 감소, 50% 감소 등)
- ✅ 실제 축소 예시 4개
- ✅ Gmail 필터링 예시: "매일 Gmail에서 특정 발신자 메일을 수신하면, 제목에 따라 3개 폴더로 자동 분류하기"

---

### Q5: 목표 (여유 시간을 어떻게 사용할까?)

**추가 요소:**
- ✅ 선택지를 상세 표 형식으로 재구성 (# | 선택지 | 설명 | 신호)
- ✅ 가드레일 분석 테이블 (선택별 사용자 상태/신호/주의사항)
- ✅ 선택별 해석 4가지 추가 (피로도/성장/비즈니스/워라밸)
- ✅ Q5의 역할 명확화 (진정한 동기, 시간 활용, 4주 목표 연계)
- ✅ 예시 대화 추가 (Python 학습 예시)

**추가된 예시 대화:**
```
Claude: "월 약 6.5시간이 절감되면, 그 시간으로 뭘 하고 싶으세요?"
사용자: "Python 공부하고 싶어요"
Claude: "정말 좋네요! 월 6.5시간이면 주당 1.5시간 정도.
         4주 내에 간단한 자동화 스크립트 정도는 학습할 수 있겠어요."
```

### Q6: 외부 연동 필요 (추가 서비스 연동이 필요한가?)

**추가 요소:**
- ✅ 선택지 표 형식 재구성 (# | 선택지 | 설명 | 영향)
- ✅ 서비스 카테고리 테이블 (8가지: 이메일/채팅/스프레드시트/데이터베이스/SNS/자동화/버전관리/클라우드)
- ✅ 각 서비스별 용도 + 난이도 명시
- ✅ "Step 2: 연동 복잡도 판단" 추가 (아니오/예 경로 분기)
- ✅ 사용자A/B 예시 대화 추가 (로컬 파일 정렬 vs Gmail+Google Sheet)
- ✅ Q6의 목표 명확화 (조기 파악, 난이도 예측, 서비스명만 수집)

**추가된 예시 대화:**
```
사용자A: "아니요, 로컬 폴더의 파일을 정렬하고 정리하는 거라서..."
→ 즉시 Plan Mode로 진행 ✅

사용자B: "네, Gmail과 Google Sheet와 연동이 필요해요"
→ 서비스명 기록 후 Plan Mode 진행 ⚠️
```
