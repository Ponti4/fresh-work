# Clarify Automations - 질문 흐름 상세 가이드

## Phase 1: 현상 파악 (Fact Check)

### Q1: 빈도 - "이 업무를 얼마나 자주 하시나요?"

**선택지:**
- 1. **매일** (주 5회 이상) → 월 20회 이상
- 2. **자주** (주 2-4회) → 월 8-16회
- 3. **가끔** (월 1-3회) → 월 1-3회
- 4. **드물게** (분기 1회 미만) → 월 1회 미만

**해석:**
- 1, 2번: ✅ 자동화 유망 (반복성 높음)
- 3번: ⚠️ 검토 필요 (반복성 낮음)
- 4번: ❌ 자동화 부담 (한 번씩 하는 게 나을 수 있음)

**다음 질문으로 진행하기 전:**
사용자 응답을 명시적으로 확인하고 반영
```
감사합니다! 매일 반복하는 업무이군요.
```

---

### Q2: 소요시간 - "한 번 할 때 보통 얼마나 걸리나요?"

**선택지:**
- 1. **5분 미만**
- 2. **30분 내**
- 3. **1시간 내**
- 4. **1시간 이상**

**해석 (Q1과 조합):**

| Q1\Q2 | 5분미만 | 30분내 | 1시간내 | 1시간이상 |
|-------|--------|--------|--------|----------|
| 매일 | 약 1.5h/월 | 약 10h/월 | 약 20h/월 | 약 20h+/월 |
| 자주 | 약 0.5h/월 | 약 5h/월 | 약 10h/월 | 약 10h+/월 |
| 가끔 | - | 약 1.5h/월 | 약 3h/월 | 약 3h+/월 |
| 드물게 | - | - | - | - |

**자동화 ROI 판단:**
- **월 5시간 이상** → 자동화 가치 높음 ✅
- **월 1-5시간** → 조건부 (Q3 규칙성 중요) ⚠️
- **월 1시간 미만** → 자동화 비추 ❌

---

### Q3: 규칙성 - "매번 거의 동일한 방식으로 하시나요?"

**선택지:**
- 예 / 아니오

**해석:**

**예 (규칙성 높음):**
- 자동화 난이도: 낮음
- 성공 확률: 높음
- 예: SNS 댓글 수집 → 매번 같은 방식 ✅

**아니오 (규칙성 낮음):**
- 자동화 난이도: 높음
- 성공 확률: 중간
- 예: 클라이언트 요청 대응 → 매번 다름 ⚠️
- 단: Q4 극단화 분석으로 핵심만 자동화 가능

**Q1-Q3 종합 판단:**
```
매일 + 30분 이상 + 규칙성 높음 = 자동화 강력 추천 ✅✅✅
매일 + 5-30분 + 규칙성 높음 = 자동화 추천 ✅✅
자주 + 30분 이상 = 조건부 추천 ✅
가끔 + 1시간 이상 = 매우 조건부 ⚠️
```

## Phase 2: 기능 분해 - 불편함부터 MVP까지

### Q3: 불편함 감지 - "이 업무에서 가장 귀찮고 힘든 부분이 뭔가요?"

**답변 방식:**
- 자유 입력 (선택지 없음)
- 예: "데이터 분류", "수동 입력", "반복되는 체크", "알림 설정"

**활용:**
- Q4의 기능 분해를 위한 맥락 제공
- 사용자가 느끼는 pain point를 그대로 포착

**다음 질문으로 진행하기 전:**
```
감사합니다! [답변] 부분이 가장 힘드시군요.
```

---

### Q4: 기능 분해 (MVP 정의) - 3-4개 반복 질문

Q3 답변을 바탕으로 **자동화할 기능을 명확히 정의**합니다.

**1번 질문: 핵심 기능**
> "그렇다면 우리가 자동화해야 할 핵심 기능은 뭘까요?"
- 예: "이메일을 자동으로 분류하기"
- 자유 입력

**2번 질문: 스코프 확정**
> "이것만 해도 충분할까요, 아니면 더 필요한가요?"
- 자유 입력 (필요시 추가 항목 확인)

**3번 질문: 복잡도 최소화**
> "더 간단하게 할 방법은 없을까요?"
- 목표: MVP 수준으로 축소
- 예: "3개 카테고리만 자동 분류하기"

**최종 확정:**
```
감사합니다! 그러면 [최종 기능]을 자동화하겠습니다.
```

## Phase 3: 가치 도출 (Value)

### Q5: 목표 - "이 문제가 완벽히 해결되어 여유 시간이 생긴다면, 무엇을 하시겠습니까?"

**선택지:**
- 1. **휴식** → 개인시간 확보
- 2. **자기계발** → 스킬 향상
- 3. **본업 집중** → 핵심 업무 투자
- 4. **칼퇴** → 워라밸

**해석 (가드레일):**

| 선택 | 의미 | 가드레일 신호 |
|------|------|------------|
| **1. 휴식** | 피로도 높음 | 긍정적 - 건강상 중요 ✅ |
| **2. 자기계발** | 성장 동기 | 매우 긍정적 - 장기 효과 ✅✅ |
| **3. 본업 집중** | 매출/영향력 증대 | 최고 긍정적 - ROI 명확 ✅✅✅ |
| **4. 칼퇴** | 워라밸 중시 | 긍정적 - 삶의 질 ✅ |

**Q5 역할:**
- 자동화의 **동기 확인**
- "절감된 시간을 어떻게 활용할 것인가?"
- 4주 목표 연결 (CLAUDE.md의 프로필과 연계)

**다음 질문으로 진행하기 전:**
```
감사합니다! 월 X시간을 [선택지]에 투자하시겠군요.
```

## Phase 4: 외부 연동 확인

### Q6: 외부 연동 필요 - "추가 서비스 연동이 필요할까요?"

**질문:**
> "이 기능을 구현하기 위해 Gmail, Slack, Google Sheet 등 **외부 서비스 연동**이 필요할까요?"

**선택지:**
- 1. **아니오** → Claude Code만으로 가능
- 2. **예** → "어떤 서비스인가요?"

**해석:**

| 선택 | 의미 | 다음 단계 |
|------|------|---------|
| **아니오** | Claude Code 기반 자동화 가능 | Plan Mode로 바로 진행 ✅ |
| **예** | 외부 API 연동 필요 | 서비스명 수집 → 추가 논의 후 Plan Mode |

**Q6의 목표:**
- 외부 API 필요 여부만 조기에 파악
- 예인 경우: 서비스명만 수집 (상세 연동은 Plan Mode에서)

**다음 단계:**
```
아니오 → Task 정의서 생성 → Plan Mode
예 → "그렇다면 [서비스]와 연동하면 되겠네요. Plan Mode에서 자세히 다룰게요."
```

## 📊 질문별 월 소요시간 자동 계산

```
기본 계산식:

빈도별 월 회수:
- 매일 → 20회 (주5일 기준)
- 자주 → 10회 (평균)
- 가끔 → 2회 (평균)
- 드물게 → 0.5회

한 번 소요시간:
- 5분미만 → 5분
- 30분내 → 20분 (평균)
- 1시간내 → 40분 (평균)
- 1시간이상 → 90분 (평균)

월 소요시간 = 월 회수 × 한 번 소요시간
```

**예시:**
```
Q1: 매일 (20회/월)
Q2: 30분내 (20분)
→ 월 소요시간 = 20회 × 20분 = 400분 ≈ 6.5시간
```
