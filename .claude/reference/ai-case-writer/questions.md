# AI 활용 사례 작성 - 질문 가이드

> 현재 세션 맥락을 기반으로 gpters 게시판용 사례글을 작성하기 위한 질문들입니다.

---

## 질문 1️⃣: 시작 계기와 불편함

### 질문
> 이 작업을 시작하게 된 계기가 무엇인가요?
> AI 사용 전에는 어떤 불편함이 있었나요?

### 가이드
- **언제부터** 이런 작업이 필요했나요?
- **무엇이 방아쇠**가 되었나요? (특정 사건, 반복된 불편함)
- **기존 방식**은 어땠나요? (소요 시간, 실수, 스트레스)

### 예시 답변
```
"매주 금요일마다 주간 리포트 정리에 2시간씩 걸렸어요.
수작업으로 데이터를 복사-붙여넣기 하다 보니 실수도 잦았고,
이번 주에도 또 같은 실수를 반복하면서 '이건 자동화해야겠다'고 생각했어요."
```

### Claude의 역할
- 현재 세션 맥락을 기반으로 **추천 답변 3개** 제시
- 사용자가 "그거로 해" 또는 직접 수정 가능
- 답변이 모호하면 꼬리질문 진행

---

## 질문 2️⃣: AI 협업 과정 - 시행착오

### 질문
> AI와 협업하면서 어떤 과정을 거쳤나요?
> 인상적이었던 순간과 막혔던 순간을 알려주세요.

### 가이드
- **사용한 AI 도구는?** (Claude Code, ChatGPT 등)
- **"오!" 했던 인상적인 순간**이 있나요?
- **막혔던 부분**은 무엇이고, 어떻게 해결했나요?
- **실제 사용한 프롬프트**를 공유해주세요 (1-2개만)

### 예시 답변
```
"Claude Code로 작업했어요.
처음엔 '주간 리포트 자동화해줘'라고 막연하게 요청했는데,
Claude가 '어떤 형식으로 출력할까요?'라고 물어봐서
기존 엑셀 양식을 보여줬더니 바로 이해하더라고요.

막혔던 건 날짜 형식이 계속 달라져서,
'YYYY-MM-DD 형식으로 고정해줘'라고 구체적으로 요청하니 해결됐어요.

가장 신기했던 건, 제가 놓친 예외 케이스를
Claude가 먼저 물어봐준 거였어요."
```

### 비개발자 친화 팁
- 코드는 최소화
- 프롬프트 전문 포함 (인용문 `>` 형식)
- 화면 캡처 권장

---

## 질문 3️⃣: 결과와 배운 점

### 질문
> 결과적으로 무엇이 달라졌나요?
> 이 과정에서 배운 점이 있나요?

### 가이드
- **Before vs After 비교** (구체적 숫자)
- **결과물**은 무엇인가요?
- **팀이나 고객 반응**은 어땠나요?
- **가장 중요한 배움**은 무엇인가요?

### 예시 답변
```
Before: 주간 리포트 2시간, 실수율 30%
After: 15분으로 단축, 실수 거의 0%

결과물: 자동화된 리포트 생성 스크립트

팀 반응: 다른 팀원들도 쓰고 싶다고 함

배운 점:
- AI에게 막연한 요청보다 구체적 예시를 주는 게 효과적
- 프롬프트는 짧게보다 상황 설명을 충분히 하는 게 좋음
- 실수를 두려워하지 말고 시도하면서 배우기
```

---

## 질문 4️⃣: 향후 계획과 참고 자료

### 질문
> 앞으로 이 결과물을 어떻게 발전시킬 예정인가요?
> 참고한 자료나 도움이 필요한 부분이 있나요?

### 가이드
- **다음 단계**는? (더 자동화, 다른 업무 적용)
- **발전 계획**이 있다면?
- **참고한 gpters 글**이나 외부 자료는?
- **도움이 필요한 부분**이 있나요?

### 예시 답변
```
다음 단계:
- 월간 리포트도 자동화 예정
- 슬랙 봇으로 만들어서 팀 전체 공유

참고 자료:
- gpters: "비개발자의 첫 Claude Code 자동화"
- Claude 공식 문서: Prompt Engineering Guide

도움 필요:
- 슬랙 연동 부분은 아직 잘 모르겠어요
```

### Claude의 제안
- 계획이 없다면 **2-3가지 아이디어 제안**
- "이런 것도 가능할 것 같아요" 형식으로 시야 확장

## ⚙️ 질문 진행 원칙

1. **한 번에 하나씩 질문**
   - 메인 질문이든 꼬리질문이든 한 메시지에 질문 하나만

2. **추천 답변 제공**
   - 각 질문 시 현재 세션 맥락을 기반으로 3개 내외 추천
   - 사용자가 "응", "그거로 해" 등으로 간단히 수락 가능

3. **서술형 자유 응답**
   - 객관식보다는 사용자가 자유롭게 서술
   - 충분한 공간 제공

4. **꼬리질문**
   - 답변이 모호하거나 더 구체적인 내용 필요 시
   - 역시 한 번에 하나씩

5. **문체 참조**
   - `claude.md`의 사용자 프로필 정보 참조
   - 사용자 상황에 맞는 톤으로 질문
