## 📋 섹션별 작성 가이드

### 1. 제목
- **형식**: `[도구명] 핵심 내용 - 임팩트`
- **예시**:
  - `[Claude Code] 주간 리포트 2시간→15분 자동화`
  - `[ChatGPT] 고객 문의 응답 템플릿 10개 30분 만에 완성`

### 2. 한줄 요약
- 작업 내용 + 결과를 1-2문장으로
- 숫자 포함 시 임팩트 증가

### 3. 이런 분들께 도움돼요
- 직군, 상황, 고민 중심으로 3개 작성
- 구체적일수록 좋음

### 4. 소개 섹션
- gpters "소개" 양식에 매칭
- Before 상황 + 계기를 구체적으로

### 5. 진행 방법 섹션
- gpters "진행 방법" 양식에 매칭
- **프롬프트 전문 필수**
- **화면 캡처 권장**
- 코드는 최소화 (비개발자 대상)

### 6. 결과와 배운 점 섹션
- gpters "결과와 배운 점" 양식에 매칭
- Before/After 숫자 비교
- 실용적인 팁 중심

### 7. 재사용 가능한 프롬프트
- 독자가 바로 복사해서 쓸 수 있게
- `[수정 필요]` 표시로 커스터마이징 포인트 명시

### 8. 도움 받은 글
- gpters "도움 받은 글" 양식에 매칭
- 링크 형식으로 제공

## 💡 작성 원칙

1. **비개발자 친화적**: 코드 최소화, 자연어 프롬프트 강조
2. **구체적 숫자**: "많이" 대신 "2시간→15분"
3. **시각 자료**: 스크린샷, 이미지 적극 활용
4. **실패도 자산**: 시행착오가 더 도움됨
5. **재사용성**: 독자가 바로 따라할 수 있게
